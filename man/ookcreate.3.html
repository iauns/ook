NAME

ookcreate - create a file to store a dataset in.  

SYNOPSIS

#include <ook.h>

struct ookfile* ookcreate(struct io interface, const char* filename
                          const uint64_t dims[3], 
                          const size_t bsize[3], 
                          enum OOKTYPE type, size_t components);
 

DESCRIPTION

ookcreate returns a struct ookfile, which is used in basically all other Ook function calls. It allows read and write access to a file.

The file is created by the call to ookcreate(). Note that the file will be truncated when opened! Ook does not currently support switching between reading and writing for a given file. However, client code may perform these in phases: e.g. all writing, then all reading, then close the file.

The interface option gives the IO interface to use; see io-interface(7). filename is the filename to read. The desired number of voxels in the dataset must be given in voxels. bsize is the desired size of the bricks in the volume. Note this must make sense for the given volume; one cannot have larger bricks than the volume itself. type is the primitive type of the data; Ook uses this and components to figure out how big each element is.

The Ook interface provides a bricked dataset, but the file this API creates is not bricked. By default, the created file will be a raw linear array with the X dimension changing fastest and the Z dimension changing slowest (i.e. the standard C array ordering). The io-interface(7) can alter this layout, of course.

 

RETURN VALUE

On success, ookcreate() returns a pointer to an opaque structure describing the file. On error, NULL is returned and errno is set appropriately.

 

ERRORS

In general, an exhaustive listing is impossible; ookcreate() can fail for any of the reasons that interface's open method can fail. Errors specific to ookcreate() are listed here.
EINVAL
Brick size is larger than volume size, or filename was invalid.
ENOMEM
No memory available to create opaque structure.
