NAME

ookwrite − write a brick into an Ook file.

SYNOPSIS

#include <ook.h>

void ookwrite(struct ookfile* of, const size_t bid,
const void* data);

DESCRIPTION

ookwrite() writes a brick into the of ookfile from the memory pointer to by data. The brick to read is described by the bid brick ID. Brick IDs start at 0 and increment until ookbricks()-1, layed out in scanline order. That is, brick ID 0 refers to the brick at coordinate {0,0,0}, brick ID 1 refers to {1,0,0}, and so on, until the maximum number of bricks in X is reached. At that point, the next brick is {0,1,0}, and the count continues in this manner. The 1-dimensional brick ID given to ookwrite() could be calculated from the 3-dimensional brick index by the equation: z*ROWS*COLS+y*COLS+x.

It is implementation-defined (see io-interface(7)) whether or not data are before an ookwrite() call returns. Other processes may see inconsistent views of the file unless they are serialized by an ookclose().

RETURN VALUE

ookwrite() does not return anything. However, should an error occur it will set errno.

ERRORS

EINVAL

The given of pointer does not point to a valid struct ookfile pointer, or data is not a valid pointer.
